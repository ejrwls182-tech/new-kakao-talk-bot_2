# '오늘 강의' 카카오톡 챗봇

제71기 신임관리자과정 16주 시간표를 바탕으로, 카카오톡에서 메시지를 보내면
강의/일정(제출 마감 포함)을 자동으로 답해주는 카카오 i 오픈빌더 챗봇용 스킬 서버입니다.

## 지원 발화
- `오늘 강의` → 오늘 일정
- `내일 강의`, `어제 강의`, `모레 강의` → 해당 날짜 일정
- `6/22 강의`, `6월 22일 강의`, `2026-07-24 강의` → 특정 날짜 일정
- `이번주 해야할일`, `이번주 할일` → 이번 주(월~일) 안에 마감인 제출/마감 항목 정리
- `다음주 해야할일`, `다음주 할일` → 다음 주(월~일) 안에 마감인 제출/마감 항목 정리
- `오늘 식단`, `내일 식단`, `6/8 식단` 등 → 해당 날짜 아침/점심/저녁 식단
- `이번주 식단`, `다음주 식단` → 해당 주(월~금) 전체 식단

## 파일 구성
- `schedule.json` : 날짜별(2026-05-25 ~ 2026-09-11) 강의/일정 데이터
- `meals.json` : 날짜별 아침/점심/저녁 식단 데이터 (update_meals.py로 갱신)
- `app.py` : Flask 스킬 서버 (오늘 날짜 기준으로 schedule.json/meals.json에서 찾아 응답)
- `update_meals.py` : NHI "주간식단" 게시판의 PDF를 받아 meals.json을 갱신하는 스크립트
- `requirements.txt` : 의존 패키지

## 5. 식단(meals.json) 갱신
NHI 사이트(https://www.nhi.go.kr/Introduce/introduce7/week/List.htm)는 매주 월요일 오전
"주간식단계획" 게시글에 PDF로 식단표를 올립니다. 이 PDF를 파싱해 `meals.json`을 갱신합니다.

### 자동 갱신
- 서버가 켜져 있는 동안, 매주 월요일 09:30(KST)에 자동으로 NHI 사이트에서 최신 PDF를 받아
  `meals.json`을 갱신하려고 시도합니다 (`app.py`의 APScheduler 작업).
- NHI 사이트가 자동 다운로드를 차단(400 Bad Request)하는 경우가 있어, 실패할 수도 있습니다.
  이때는 아래 "수동 갱신"을 사용하세요.
- 자동 갱신 비활성화: 환경변수 `DISABLE_MEAL_SCHEDULER=1`
- 원격으로 즉시 갱신 트리거: 환경변수 `ADMIN_TOKEN` 설정 후
  `POST /admin/update-meals` (헤더 `Authorization: <ADMIN_TOKEN>`)

### 수동 갱신
1. NHI 홈페이지 → "주간식단" 게시판에서 최신 PDF 다운로드
2. 아래 명령 실행:
   ```
   python update_meals.py --pdf "주간식단계획(6.8~6.12).pdf"
   ```
3. PDF의 표 형식이 예상과 다르면 먼저 `--dry-run`으로 추출 결과를 확인:
   ```
   python update_meals.py --pdf "주간식단계획(6.8~6.12).pdf" --dry-run
   ```
   결과가 이상하면 `update_meals.py`의 `parse_table_to_meals()` 함수를 PDF의 실제
   표 구조(헤더 날짜 표기, 조식/중식/석식 행 이름 등)에 맞게 조정하세요.

## 1. 서버 배포 (예: Render.com 무료 플랜)
오픈빌더가 스킬을 호출하려면 **인터넷에서 접근 가능한 HTTPS URL**이 필요합니다. 로컬 PC만으로는 안 됩니다.

1. GitHub에 이 폴더(app.py, schedule.json, requirements.txt)를 올립니다.
2. Render.com → New → Web Service → 해당 GitHub repo 연결
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn app:app`
3. 배포 완료 후 발급되는 주소 확인 (예: `https://my-bot.onrender.com`)
4. `https://my-bot.onrender.com/health` 접속 시 `{"status":"ok"}` 가 나오면 정상

(Render 대신 Railway, Fly.io, 자체 서버 등 어디든 Flask 앱을 띄울 수 있는 곳이면 됩니다.)

## 2. 카카오 i 오픈빌더 설정
1. https://kakaobusiness.kakao.com 또는 https://chatbot.kakao.com 에서 카카오톡 채널 + 챗봇 생성
2. 좌측 메뉴 **스킬(Skill)** → 스킬 추가
   - 이름: `오늘강의`
   - URL: `https://my-bot.onrender.com/skill`
3. **시나리오 → 블록 추가**
   - 블록 이름: `강의일정`
   - 사용자 발화(트리거): `오늘 강의`, `내일 강의`, `이번주 해야할일` 등 원하는 표현 추가
     - 특정 날짜 발화는 패턴 매칭(예: `{날짜} 강의`)으로 한 블록에 등록하면 다양한 날짜 표현을 한 번에 받을 수 있습니다.
   - 스킬 연결: 위에서 만든 스킬 선택 (스킬 데이터 사용 → 봇 응답으로 출력)
   - 서버가 발화 텍스트(`userRequest.utterance`)를 직접 분석해 날짜/이번주 여부를 판단하므로,
     여러 발화를 같은 블록·같은 스킬에 연결해도 됩니다.
4. 챗봇 배포(운영 채널로 전환) 후, 해당 카카오톡 채널을 친구 추가하고 "오늘 강의"를 보내면
   서버가 응답하는 메시지가 옵니다.

## 4. 데이터 수정/추가
`schedule.json`의 날짜별 배열에 문자열을 추가/수정하면 됩니다. (날짜 형식: `YYYY-MM-DD`)
교육 기간(5/25~9/11) 외 날짜는 "교육 기간에 포함되지 않은 날짜입니다."로 응답합니다.

## 참고
- 시간대는 한국시간(Asia/Seoul) 기준으로 "오늘"/"이번주"/"다음주"를 계산합니다.
- "이번주 해야할일"/"다음주 해야할일"은 "제출" 또는 "마감"이 포함된 항목만 골라 보여줍니다.
  새로운 마감 항목을 `schedule.json`에 추가할 때 해당 표기를 포함시켜야 인식됩니다.
- 각 강의 항목에는 가능한 경우 시작~종료 시간을 표기했습니다. 원본 시간표에 시간이 명시되지 않은
  강의는 일반적인 수업 시간(10:00~13:00, 14:00~17:00)을 기준으로 추정해 넣었으므로,
  실제 시간과 다르면 `schedule.json`에서 해당 줄의 시간을 직접 수정하면 됩니다.

## 오픈빌더에서 "6/22 강의" 같은 발화가 인식되지 않을 때
@sys.date 같은 시스템 엔티티는 "6/22", "6월 22일", "다음주 월요일" 같은 표현을 항상
인식하지 못해 폴백("이해하기 어려워요")으로 빠질 수 있습니다. 이 경우 블록의 패턴 발화에
`*강의`, `*할일`, `*할 일` 같은 와일드카드 패턴을 추가하면, "강의"/"할일"로 끝나는
모든 문장이 스킬로 전달되어 서버가 직접 날짜를 해석합니다(서버 로직은 이미 모든 형식을 지원).
